Ⅴ. 컴퓨팅 사고 Ⅴ

1. 순서열- 계산 자료의 ‘더미’

계산 자료 = 같은 꼴의 더미

  • 같은 꼴의 더미로 만들어주어 컴퓨터가 계산하게 함

순서열

  • 같은 꼴의 더미를 표현
    • [계산자료1, 계싼자료2, …, 계산자료n]
  • 예
    • []
    • [1]
    • [1,2,2,3,3,3]
    • [‘월요일’, ‘화요일’, ‘수요일’]

다음 유도하기로 더미 해석하기

  • 최초 빈 더미
    • []
  • 직전 더미에 원소 하나 추가
    • [1,2,3] + [100] = [100,1,2,3] (맨 앞에 덧붙여짐)

다음 유도하기에 맞춰 순서열 만들기

  • 해당 기능을 바로 쓸 수 없고, 다른 기능을 이용해 표현
  • 빈 순서열
    • []
  • 작은 순서열 맨 앞에 원소 하나를 덧붙여 얻은 순서열
    • [원소] + 작은 순서열
      • [1] == [1] + []
      • [2,3] == [2] + [3]
      • [1,2,3] == [1] + [2,3]

순서열과 여러곱

  • 여러곱
    • 원소 개수가 고정
    • 원소를 추가하면 새로운 꼴의 여러곱이 나옴
      • 서울 그리고 대전
      • 서울 그리고 대전 그리고 부산
  • 순서열
    • 원소가 몇 개 포함될지 미정
    • 원소를 추가해도 종류가 달라지지 않음
      • 서울, 대전 더미
      • 서울, 대전, 부산 더미

Python 언어에서 여러합 원소 더미

  • pythoh에서 여러합을 바로 나타낼 수 없음
  • 예: 이름 또는 생년월일 더미
    • 이름 홍길동, 생년월일 1981년 9월 18일 더미
    • [(‘이름’,‘홍길동’),(‘생년월일’,(1981,9,18))]
    • 같은 꼴의 더미를 표현하는 순서열도 실제 원소 꼴은 다를 수 있음

순서열을 쓸 수 있는 곳

  • 기본 자료가 나올 수 있는 곳 전부
    • 예: 여러곱 원소, 순서열 원소

해석

  • 원소 두 개 여러곱 = 대응 관계
    • 대응 관계 해석
      • (계산자료1, 계산자료2) ⇔ 계산자료1은 계산자료2에 대응된다,
      • (‘인천’, 40) ⇔ 인천은 40km에 대응된다. / 인천 특징은 40km이다.
      • (‘서울’, [(‘인천’,40),(‘대전’,163),(‘강릉’,237)])
      • ⇔ 서울 특징으로 인천은 40km에 대응되고 대전은 163km에 대응되고 강릉은 237km에 대응된다.

정리

  • 계산 자료
    • 기본 자료: 정수, 논리값, 문자열
    • 여러곱: - 그리고 -, -에 대응된다
    • 여러합: - 또는 -
    • 순서열: - 더미

2. 다음 유도하기로 계산 자료 만들기

다음 유도하기와 여러합

  • 다음 유도하기에서 계산 자료
    • ‘최초 경우’ 또는(여러합) ‘작은 것으로부터 더 큰 것을 이끌어 내는 경우’
      • 여러합 원소 = (종류, 내용)
      • Python 언어에서 만들어 낼 수 있음

계산 자료 조합하기

  • 기본 자료: 정수, 논리값, 문자열
  • 여러곱: - 그리고 -, -에 대응된다
  • 여러합: - 또는 -
  • 순서열: - 더미
    • 다음 유도하기로 정의하기

Ⅵ. 컴퓨팅 사고 Ⅵ

1. 연산

계산 = 무엇 + 어떻게

  • 계산 = 무엇(계산 대상) + 어떻게(계산 과정)
    • 계산 대상: 알맹이 요약하기, 계산 자료
    • 계산 과정: 연산, 명령

컴퓨터 암산

  • 계산 자료 처리하여 계산 자료 만들기
    • 정수 1729와 91을 더한다
    • 정수 1729 → “더한다” ← 정수 91
    • 정수 1729(목적어) = 1729(피연산자)
    • “더한다”(동사) = +(연산자)

연산자 종류

  • 단항 연산자
    • <단항 연산자><피연산자>
      • 1729
  • 이항 연산자
    • <피연산자1><이항 연산자><피연산자2>
    • 1729 + 91
  • 고유 문법이 있는 연산자

Python 언어 연산자

  • 정수 연산자
    • 이항 연산자: 덧셈, 뺄셈, 곱셈, 나눗셈, 나머지 지수승
    • 단항 연산자: 부호 그대로 두기 +, 부호 바꾸기 -
  • 논리값 연산자
    • 이항 연산자: 논리곱 and, 논리합 or
    • 단항 연산자: 논리 부정 not
  • 문자열 연산자
    • 이항 연산자 | 잇기 +

2. 연산식

연산식

  • 연산을 표현하는 프로그램

Python 언어 연산자 우선 순위

  • 우선 순위: 연산자

    • 지수승 **
    • 부호 그대로 두기 +, 부호 바꾸기 -
    • 곱셈 *, 나눗셈 /, 나머지 %
    • 덧셈 +, 뺄셈 -, 문자열 잇기 +
    • 논리 부정 not
    • 논리곱 and
    • 논리합 or

  • 논리값 계산하는 이항 연산자

    • 왼쪽으로 결합
    • 같은지 판별
      • ==: 피연산자가 똑같으면 True 계산
      • !=: 피연산자가 서로 다르면 True 계산
    • 대소 판별
      • 수학에서와 의미 같음
      • 정수, 논리값, 문자열 비교

비교 연산자와 논리 연산자 결합하기

  • 비교 연산자 우선 순위
    • 논리 연산자보다 높음
    • 그 외 연산자보다는 낮음
  • 괄호를 쓰지 않아도 괜찮을 때가 많음

Ⅶ. 컴퓨팅 사고 Ⅶ

1. 상수- 컴퓨터 쪽지

  • 이름이 있음
  • 다시 덮어 쓸 수 있음
    • 항상 같은 값만 읽음
    • 읽을 때 외부와 소통하지 않음
    • 연산
  • 버리는 과정은 따로 없음

Python 언어에서 상수

  • 변수의 특수한 경우로 흉내냄
    • 이름 = 연산식
  • 상수 이름
    • 영어 알파벳, 숫자, 특수 문자 _ 등을 조합
    • 첫 글자는 숫자가 될 수 없음
    • 컴퓨팅 환경에 따라 한글을 쓸 수 있음
  • 상수 이름을 잘 정하는 것이 중요함

어떤 계산할지 판단하기

  • 다음 유도하기
    • 만약 쪽지에 적힌 수가 0이면 합으로 0을 계산하고 아니면 쪽지에 적힌 수의 직전 수까지 합을 구한 뒤 쪽지에 적힌 수를 다시 더한다.
  • 쪼개 풀어 합치기
    • 만약 생활기록부가 하나만 남았다면 남은 생활기록부에서 성적을 구하고 아니면 생활기록부 더미를 둘로 쪼개서 각 더미에서 성적 합을 구한 뒤 더한다.

Python 언어 여러곱 연산자

  • 여러곱으로부터 원소를 뽑아 냄

    • 여러곱 연산식[정수 연산식]

  • 여러곱 연산자: ‘그리고’ 의미에 맞게 빠뜨림 없이

    • 그리고의 의미에 맞게 작동
    • 그리고로 엮은 계산 자료를 빠뜨림 없이 처리

2. 여러합 연산자

여러합 연산자

  • '또는’의 의미에 맞게 빠뜨림 없이
    • 또는의 의미에 맞게 작동
    • 또는으로 정의한 여러합 원소를 빠뜨림 없이 처리

Ⅷ. 순퓨팅 사고Ⅷ

1. 순서열 연산자

순서열 연산자

  • '더미’의 의미에 맞게 빠뜨림 없이
    • '더미’의 의미에 맞게 작동
    • '더미’로 정의한 순서열 원소를 빠뜨림 없이 처리

순서열 연산자 설계 검증

  1. 순서열을 만든 직후, 원소나 원소 더미를 뽑아내어 검사
  2. 내용을 모르는 순서열을 받아서 복제할 수 있는지 검사

순서열 연산자 설계

  • 다음 유도하기에 맞추어 순서열 만들기
    • 빈 순서열: []
    • 작은 순서열 맨 앞에 원소 하나를 덧붙여 얻은 순서열: [원소] + 작은 순서열
  • 필요한 연산자
    1. 두 가지를 구분하는 연산자
    2. 빈 순서열이 아닐 때, 맨 앞 원소와 작은 순서열을 계산하는 연산자
      • 순서열을 빈 순서열과 비교: 순서열 == []
      • 순서열 맨 앞 원소 계산: 순서열[0]
      • 순서열 맨 앞 원소 이후의 작은 순서열 계산: 순서열[1:]

순서열 복제하기

  • 사용하는 연산자
    • 순서열 == []
    • 순서열[0]
    • 순서열[1:]
  • 상수 op에 기록된 순서열 복제하기
    • [] if op == [] else [op[0]] + op[1:]

하나로 통합한 순서열 연산자

  • 상수 op에 순서열이 기록되어 있을 때
    • (빈 순서열일 때 연산식 if op == [] else
      lambda hd, tl: 빈 순서열이 아닐 때 연산식)(op[0], op[1:]))
      • hd = head
      • tl = tail
